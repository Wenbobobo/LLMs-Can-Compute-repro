# F19 Acceptance

- the semantic surface is bounded explicitly rather than left rhetorical;
- `R42`, `R43`, and `R44` each have fixed preconditions, outputs, and stop
  rules;
- useful kernels are fixed in order and do not require implementer-side route
  choices;
- arbitrary `C`, heap, float, IO, and alias-heavy pointers remain excluded;
- no runtime lane is activated by `F19`.
