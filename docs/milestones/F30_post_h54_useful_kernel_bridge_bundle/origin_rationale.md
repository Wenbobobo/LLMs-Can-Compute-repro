# Origin Rationale

This successor packet is derived from the local-only origin materials under
`D:/zWenbo/AI/LLMCompute/docs/origin/` plus the current landed repo evidence.

The common conservative reading across the field note and the three saved
discussions is:

- the headline "LLMs are computers" is broader than the publicly verifiable
  evidence;
- the strongest credible core is an append-only execution substrate with
  structured retrieval and a narrow exact executor;
- broad `C`, broad Wasm, trainable entry, transformed entry, and general
  systems claims are downstream and should not be reopened by momentum; and
- the smallest meaningful falsifier after a toy compiled-boundary success is
  whether the same route carries any preserved useful kernel without hidden
  compiler-side work.

That is why this bundle does not reopen broader frontend growth and does not
route back to `F27/R53/R54`. It stores only one narrow question:

can the exact compiled-boundary route proven by `R58/R59` carry a minimal
useful-kernel family that already exists elsewhere in the repo?

If the answer is no, the compiled-boundary route should likely be treated as a
toy exactness result. If the answer is yes, the project still needs a separate
value gate before claiming that the route matters beyond mechanistic support.
