# P9 Release Candidate and Public Surface Freeze Design

## Context

Once `P8` and `H2` hold, the remaining task is a restrained outward sync. This
should freeze the public surface around the current paper-grade endpoint rather
than turn into a broader narrative wave.

## Goal

Move the repo landing page, status page, and short release-facing text into a
release-candidate state that is fully downstream of the locked manuscript
bundle and explicit about what remains blocked.

## Required Outputs

- one milestone scaffold under
  `docs/milestones/P9_release_candidate_and_public_surface_freeze/`;
- one release-candidate checklist for README / STATUS / release-summary / blog
  gating;
- one handoff for restrained outward sync work;
- synchronized public-facing docs that still preserve the blocked-blog rule.

## Acceptance

- README, STATUS, and release summary speak with one paper-first voice;
- the current endpoint remains narrow and explicit;
- blog work stays blocked unless the release-candidate checklist and existing
  blog rules are both satisfied.
