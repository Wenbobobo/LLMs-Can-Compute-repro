# R4 Mechanistic Retrieval Closure Design

## Goal

Close the mechanistic story on the same positive `D0` suites by showing that
successful execution is carried by a small fixed set of retrieval primitives
rather than by an opaque executor claim.

## Intended outputs

- one `R4` milestone scaffold;
- one bridge export mapping each positive `D0` dependency to latest-write,
  stack, control, or deterministic local-transition classes;
- one agreement export showing source-event parity between `linear` and `Hull`
  retrieval on the same rows;
- one diagnostic companion that reuses staged-pointer / provenance artifacts
  only as mask-strength evidence.

## Scope lock

- current positive `D0` suites only;
- no rescue work on `M5` baselines by default;
- no widening of the neural branch beyond what the same mechanistic story
  requires;
- no systems claim changes in this lane.

## Acceptance

- each positive suite is explainable in the allowed primitive set;
- `linear` and `Hull` retrieval agree at the source-event level;
- any unexplained dependency is recorded explicitly as a contradiction
  candidate rather than hidden in broader prose.
