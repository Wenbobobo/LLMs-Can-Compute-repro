# P7 Manuscript Freeze Candidate and Release Preflight Design

## Context

`P6` closed the local layout/readiness pass without reopening scientific scope.
The manuscript bundle now has stable local placement choices, the public
surface remains downstream of that bundle, and the narrow `P5` audits still
guard against scope drift. The next paper-facing wave is therefore not another
layout cleanup pass. It is a freeze-candidate and release-preflight pass.

## Goal

Move the current paper bundle from “internally stable markdown draft” to “clear
freeze candidate with an explicit outward release checklist,” while keeping blog
work blocked unless the release-preflight conditions are satisfied.

## Required outputs

- one milestone scaffold under
  `docs/milestones/P7_manuscript_freeze_candidate_and_release_preflight/`;
- explicit freeze-candidate criteria for the current manuscript bundle;
- one release-preflight checklist covering README / STATUS / release summary /
  paper-facing ledgers;
- one final figure/table order and appendix-companion inclusion plan;
- one note stating what must remain true before any future blog derivative is
  allowed to move.

## Acceptance

- a later plan-mode pass can start from `P7` without reopening `P6`;
- the freeze candidate is defined by explicit criteria rather than by vague
  prose quality judgments;
- outward release work remains downstream of the frozen claim/evidence bundle.
