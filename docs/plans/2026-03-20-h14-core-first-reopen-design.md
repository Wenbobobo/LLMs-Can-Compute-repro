# H14 Core-First Reopen Design

## Goal

Reopen the post-`H12` stage on the narrow mechanism path rather than leaving
the repo parked indefinitely in governance-only rollover.

## Options

### Option A: core-first reopen with explicit scope lock

Make `H14` the active driver, keep `H13/V1` preserved as handoff state, and
route the next evidence wave through exact geometry plus append-only/latest-write
execution before any bounded compiled follow-up.

Recommendation: use this option. It reopens real scientific work without
throwing away the control chain built in `H13/V1`, and it matches the
repository's actual strongest substrate: exact 2D retrieval plus append-only
execution on the same fixed endpoint.

### Option B: keep the repo frozen in governance-only mode

Leave `H13/V1` active until a later human-planned packet is ready.

Not recommended. That avoids risk, but it also stalls the reproduction-first
goal and leaves the repo in a holding pattern after the control handoff is
already complete.

### Option C: jump directly to a compiled or demo-facing follow-up

Treat the reopened lane as a chance to push compiled-language or presentation
surface claims immediately.

Not recommended. The present evidence does not justify that jump, and doing so
would widen faster than the mechanistic substrate has been reclosed.

## Intended outputs

- one saved reopened-stage master plan in `tmp/`;
- one `H14` milestone scaffold that locks the reopened scope;
- concrete `R11/R12/R13/R14/H15` milestone scaffolds and plan docs;
- one machine-readable `H14` stage guard;
- minimal root/publication/control-surface updates that switch the active
  driver from `H13/V1` to `H14` while preserving the handoff.

## Scope lock

- do not reinterpret `H13/V1` as failed; preserve it as completed handoff;
- do not treat `V1` runtime classification as scientific evidence;
- do not let `R14` widen beyond tiny typed bytecode `D0`;
- do not weaken `M7` no-widening or downstream-only derivative-writing rules.

## Acceptance

- the canonical driver names `H14` as active and `H13/V1` as preserved handoff;
- the reopened lane has explicit execution order and bounded follow-up tracks;
- historical guards and standing audits remain green after the stage switch.
