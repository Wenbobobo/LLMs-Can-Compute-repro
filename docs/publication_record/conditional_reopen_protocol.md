# Conditional Reopen Protocol

State: `archive_first_dormant_protocol`.

This file defines the only admissible way to reopen science after the current
archive-first consolidation wave.

## Current rule

No runtime lane is open from the current archive stack.

This protocol does not itself authorize runtime. It only defines the minimum
conditions under which a later explicit authorization packet may be proposed.

no runtime lane opens from this protocol by itself.

## Single admissible future family

Only one future route remains admissible on paper:

- `compiled_online_exact_retrieval_primitive_or_attention_coprocessor_route`

All same-lane executor-value microvariants remain inadmissible. `F27`, `R53`,
and `R54` remain blocked.

## Entry conditions

A later reopen packet is admissible only if all of the following are true:

1. the question cannot be answered by archive, wording, indexing, or existing
   result packets alone;
2. the proposal stays at the exact useful-kernel contract or a strictly
   narrower transparent target;
3. the proposal names a materially different cost structure relative to the
   closed `R62/H58` executor-value lane;
4. the proposal declares transparent linear and external/simple baselines; and
5. the proposal is self-contained enough to be judged before runtime.

## Qualification-only dossier

Before any later runtime authorization, the proposer must provide a
qualification-only dossier that names:

- the exact useful-case target;
- the claimed cost-model change;
- the complete baseline set;
- the bounded-value win condition; and
- the stop rules that immediately terminate the lane if the route collapses
  back into renamed same-lane work.

This is the `F36` standard.

## Immediate non-triggers

- wording drift alone is not a valid reopen trigger;
- curiosity-driven widening is not a valid reopen trigger;
- branch hygiene or merge convenience is not a valid reopen trigger;
- arbitrary `C`, broader Wasm, transformed entry, and trainable entry are not
  valid reopen triggers; and
- "attention might help" without a concrete cost-model delta is not a valid
  reopen trigger.

## Refreeze requirement

If a later explicit authorization packet is ever granted, the follow-up must:

1. stay inside the declared narrow target and baseline contract;
2. stop immediately if the qualification rules fail;
3. rerun the relevant audits; and
4. return control to the archive-first stack unless a later explicit packet
   raises the ceiling.

No runtime lane opens from this protocol by itself.
